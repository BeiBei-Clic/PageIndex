\chapter{实验目的和背景}
\label{chap:background}

\section{实验目的}

本次课程作业的主要目的是设计并实现一个能够记录Python程序执行路径和覆盖率的工具。具体目标包括：

\begin{enumerate}
    \item 掌握Python运行时追踪技术，理解\texttt{sys.settrace}的工作原理
    \item 学习使用抽象语法树（AST）进行静态代码分析
    \item 能够识别程序中的基本控制流结构：顺序、选择（if/elif/else）和循环（for/while）
    \item 记录程序运行时的执行路径，包括函数调用关系
    \item 计算代码覆盖率并生成可读的执行分支报告
\end{enumerate}

\section{实验背景}

\subsection{代码覆盖率概述}

代码覆盖率是软件测试中的重要指标，用于衡量测试用例对被测代码的覆盖程度。常见的覆盖率类型包括：

\begin{itemize}
    \item \textbf{语句覆盖}：测量被执行的代码语句比例
    \item \textbf{分支覆盖}：测量控制流分支的执行比例（如if语句的true/false分支）
    \item \textbf{路径覆盖}：测量程序执行路径的覆盖比例
\end{itemize}

本实验主要关注\textbf{分支覆盖}，即识别程序中的控制流分支点，并统计在运行时被触发的分支数量。

\subsection{Python追踪机制}

Python提供了\texttt{sys.settrace()}函数，允许注册一个回调函数来监控程序的执行。当程序执行时，Python解释器会在以下事件发生时调用回调函数：

\begin{itemize}
    \item \texttt{'call'}：函数被调用时
    \item \texttt{'line'}：执行新的一行代码时
    \item \texttt{'return'}：函数返回时
    \item \texttt{'exception'}：发生异常时
\end{itemize}

利用这一机制，可以精确记录程序的执行路径。

\subsection{AST静态分析}

Python的\texttt{ast}模块可以将源代码解析为抽象语法树（AST）。通过遍历AST，可以识别程序中的所有控制流结构：

\begin{itemize}
    \item \texttt{ast.If}：if条件语句
    \item \texttt{ast.For}：for循环语句
    \item \texttt{ast.While}：while循环语句
    \item \texttt{ast.ExceptHandler}：异常处理器
\end{itemize}

\section{作业要求}

根据作业要求，本工具需要满足以下功能：

\begin{enumerate}
    \item 记录Python程序执行路径和覆盖率
    \item 分析基本程序常见结构流：选择、顺序与循环
    \item 记录程序运行执行路径
    \item 生成描述程序运行执行分支的报告
\end{enumerate}

程序结构需要清晰，代码需要有必要的注释，并提供使用文档。
