## 二、技术原理

### 2.1 运行时追踪技术

本实验使用Python的`sys.settrace`机制实现运行时追踪。该机制允许注册一个回调函数，在程序执行过程中，每当发生以下事件时，回调函数会被调用：

- `call`：函数调用时触发
- `line`：执行新的一行代码时触发
- `return`：函数返回时触发
- `exception`：发生异常时触发

核心追踪代码如下：


### 2.2 静态控制流分析

使用Python的`ast`模块进行静态分析，识别代码中的控制流结构：

```python
def _extract_branch(self, file_path: str, node: ast.AST):
    if isinstance(node, ast.If):
        return CodeBranch(file_path, node.lineno, BranchType.IF)
    elif isinstance(node, ast.For):
        return CodeBranch(file_path, node.lineno, BranchType.FOR)
    elif isinstance(node, ast.While):
        return CodeBranch(file_path, node.lineno, BranchType.WHILE)
    elif isinstance(node, ast.ExceptHandler):
        return CodeBranch(file_path, node.lineno, BranchType.EXCEPT)
```

### 2.3 覆盖率计算

覆盖率 = (已执行分支数 / 总分支数) × 100%

### 4.4 代码示例：追踪demo.py

```python
from pathtracer import PathTracer, CoverageReporter

# 创建追踪器并指定目标文件
tracer = PathTracer().trace_file("demo.py")

# 开始追踪
with tracer:
    # 在此执行目标代码
    demo_conditionals(25)
    demo_for_loop([1, 2, 3, 4, 5])
    demo_while_loop(5)
    demo_exception_handling(10, 0)

# 生成覆盖率报告
reporter = CoverageReporter()
report = reporter.analyze("demo.py", tracer)
print(reporter.format_report(report))
```

## 六、结果分析

### 6.1 覆盖率分析

agent_pageindex.py的覆盖率为31.8%，主要原因：

1. **verbose模式未触发**：代码中`_print_verbose_stream`函数（40-75行）未被调用，因为测试时未使用`--verbose`参数
2. **异常分支未覆盖**：API密钥存在性检查的异常分支（89行）未触发
3. **复杂消息处理未执行**：`_extract_text`函数中的列表处理分支未完全覆盖
4. **边缘条件**：部分边缘条件的处理逻辑未在正常流程中执行

### 6.2 改进建议

1. **增加测试用例**：
   - 添加`--verbose`参数测试
   - 模拟无API密钥场景
   - 测试不同类型的消息内容

2. **提高覆盖率**：
   ```bash
   # 测试verbose模式
   python -m pathtracer.cli agent_pageindex.py -- --query "test" --verbose

   # 测试不同参数组合
   python -m pathtracer.cli agent_pageindex.py -- --query "test" --doc-top-k 5
   ```

3. **单元测试补充**：为各函数编写独立的单元测试，确保所有分支都被覆盖

## 七、结论

本实验成功实现了一个Python程序执行路径和覆盖率分析工具（PathTracer），具备以下功能：

1. **运行时追踪**：使用`sys.settrace`记录程序执行的每一行代码
2. **静态分析**：使用AST识别代码中的所有控制流分支
3. **覆盖率计算**：对比静态分支与动态执行路径，计算覆盖率
4. **多格式报告**：支持文本、JSON、HTML三种报告格式
5. **函数调用树**：记录并展示函数调用关系、参数和返回值

该工具能够有效分析Python程序中的选择、顺序与循环等基本结构流，生成详细的执行分支报告，满足作业要求。
