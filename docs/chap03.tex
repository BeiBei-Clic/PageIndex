\section{测试目标程序}

\subsection{程序简介}

本实验使用\texttt{agent\_pageindex.py}作为测试目标程序。该程序是一个基于LangChain的文档问答代理，共130行代码，具有以下特点：

\begin{itemize}
    \item 多层条件判断（参数验证、类型检查、消息处理）
    \item 循环结构（消息处理、工具调用遍历）
    \item 函数调用（包含嵌套调用和递归调用树）
    \item 异常处理（API密钥检查、异常捕获）
\end{itemize}

\subsection{覆盖率合理性说明}

实验结果显示demo.py覆盖率为71.4\%，agent\_pageindex.py最高为72.7\%。这个覆盖率是合理的，原因如下：

\begin{enumerate}
    \item \textbf{防御性代码占比较高}：未覆盖的6个分支中，有2个是空值检查和异常抛出，属于防御性编程
    \item \textbf{输入类型相关分支}：2个分支依赖特定的list类型输入，当前测试用例使用str类型
    \item \textbf{逻辑互斥}：1个分支与另一分支逻辑互斥（126行与124行）
\end{enumerate}

因此，72.7\%的覆盖率反映的是"有效业务路径已较充分覆盖"，而非测试设计不足。

\subsection{实验对象说明}

本实验主要使用demo.py和agent\_pageindex.py作为测试对象，原因如下：

\begin{enumerate}
    \item demo.py是专门设计的测试程序，包含所有控制流结构（if/for/while/except/嵌套）
    \item agent\_pageindex.py是真实项目代码，用于验证工具在复杂场景下的表现
\end{enumerate}

\textbf{局限性}：本实验未对大规模项目（如Django、Flask框架）进行测试，因此工具在大规模代码库上的表现有待进一步验证。

\subsection{控制流图示例}

PathTracer识别的控制流结构示例（demo.py \texttt{demo\_conditionals}函数）：

\begin{lstlisting}[language=Python, caption=控制流结构示例]
def demo_conditionals(value: int) -> str:
    if value < 0:           # Branch 1: IF (line 32)
        return "negative"
    elif value == 0:      # Branch 2: IF (line 34)
        return "zero"
    elif value <= 10:      # Branch 3: IF (line 36)
        return "small"
    elif value <= 50:      # Branch 4: IF (line 38)
        return "medium"
    else:                 # (implicit else)
        return "large"
