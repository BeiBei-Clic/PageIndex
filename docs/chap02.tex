\subsection{模块划分}

系统主要包含以下模块：

\begin{itemize}
    \item \textbf{models.py}：数据模型定义，包含分支类型枚举、代码分支、函数调用、执行路径和覆盖率报告等数据类
    \item \textbf{tracer.py}：运行时追踪器，使用\texttt{sys.settrace}记录程序执行路径
    \item \textbf{cfg\_builder.py}：控制流图构建器，使用AST静态分析识别所有分支点
    \item \textbf{reporter.py}：报告生成器，计算覆盖率并生成多种格式报告
    \item \textbf{cli.py}：命令行接口，提供用户友好的使用方式
\end{itemize}

\subsection{核心数据类}

主要数据类包括：

\begin{itemize}
    \item \texttt{CodeBranch}：表示代码中的一个分支点，包含文件路径、行号、分支类型、结束行号、是否已执行等信息
    \item \texttt{FunctionCall}：记录函数调用信息，包含函数名、参数、返回值、调用深度、子调用等
    \item \texttt{ExecutionPath}：记录执行路径，包含已执行行号集合、已执行分支、函数调用列表
    \item \texttt{CoverageReport}：覆盖率报告，包含总分支数、已执行分支数、按类型统计、覆盖率百分比等
\end{itemize}

\section{运行时追踪器实现}

\subsection{追踪回调函数}

核心追踪逻辑通过\texttt{\_trace\_callback}方法实现，在程序执行过程中自动记录：

\begin{enumerate}
    \item \textbf{line事件}：记录执行的代码行号
    \item \textbf{call事件}：记录函数调用信息（函数名、参数、调用深度）
    \item \textbf{return事件}：记录函数返回值
\end{enumerate}

\subsection{上下文管理器}

使用Python的上下文管理器协议（\texttt{\_\_enter\_\_}/\texttt{\_\_exit\_\_}）简化追踪的启动和停止：

\begin{lstlisting}[language=Python]
tracer = PathTracer().trace_file("target.py")
with tracer:
    # 在此执行目标代码
    result = target_function()
# 退出with块后自动停止追踪
\end{lstlisting}

\section{控制流图构建器}

\subsection{AST遍历}

使用\texttt{ast}模块遍历源代码的抽象语法树，识别控制流节点：

\begin{lstlisting}[language=Python]
def _extract_branch(self, file_path, node):
    if isinstance(node, ast.If):
        return CodeBranch(file_path, node.lineno,
                          BranchType.IF)
    elif isinstance(node, ast.For):
        return CodeBranch(file_path, node.lineno,
                          BranchType.FOR)
    elif isinstance(node, ast.While):
        return CodeBranch(file_path, node.lineno,
                          BranchType.WHILE)
\end{lstlisting}

\section{报告生成器}

\subsection{覆盖率计算}

报告生成器通过对比静态分支与执行路径计算覆盖率：

\begin{equation}
    \label{eq:coverage}
    \text{覆盖率} = \frac{\text{已执行分支数}}{\text{总分支数}} \times 100\%
\end{equation}

\section{本章小结}

本章详细介绍了PathTracer系统的设计与实现。系统采用模块化架构，分为数据模型、运行时追踪、静态分析、报告生成四个核心模块。通过Python标准库提供的\texttt{sys.settrace}和\texttt{ast}模块，实现了对Python程序执行路径的完整追踪和覆盖率分析功能。
